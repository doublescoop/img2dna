img2dna

Encode a black-and-white image as DNA, decode DNA back to an image. Entirely client-side.

W=image width · GR_K=5 · MARKER=TCCG · FSM map (see below) · per-strand header: guard(1b) + salt(16b)

FSM map — how bits become bases (and back)

Bits are grouped into base-3 digits (trits) and each trit picks the next DNA base according to the previous one. The rule guarantees no base is ever repeated (no AA/CC/GG/TT) and keeps GC% balanced.

previous basetrit = 0trit = 1trit = 2

First base (no previous): trit 0 → A, trit 1 → C, trit 2 → G. To decode, look up the row for the previous base and read the column whose cell equals the current base — that column index is the trit.

How decoding works (index-card spec)
1. Keep only A/C/G/T (uppercase). 2. Split on TCCG — each piece is one chunk. 3. For each piece: a. Convert DNA → trits using the FSM b. Trits → bits (base-3 to binary) c. Skip leading zeros; first '1' is the guard d. Next 16 bits = salt e. XOR the rest with salt (repeating every 16 bits) f. Loop: read 1 bit = row start color; GR-decode runs until they sum to W pixels; paint that row with alternating colors 4. Paint rows top-to-bottom on a white canvas. Undecodable chunks are skipped. Unpainted pixels stay white.

Decoded image

Rows paint in the order they appear in the pasted text. Drop chunks, rearrange, or paste partial DNA — only the rows you have will render.
